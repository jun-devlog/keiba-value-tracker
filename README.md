# keiba-value-tracker

競馬データ分析・収支管理アプリ

自身の競馬予想のプロセスや過去の成績（収支・回収率）を記録し、客観的に分析・管理するためのアプリケーションです。
現在、バックエンドAPIとReactフロントエンドからなる **MVP（Minimum Viable Product）** が完了しており、一連の競馬収支管理フローをブラウザ画面から操作可能な状態になっています。

## アプリの目的
自身の予想とその結果を一元管理し、振り返りを行いやすくすることを目的としています。

> [!WARNING]
> **対象外の機能**  
> 本アプリはあくまで「データの記録・分析」を目的としています。以下のような機能は**対象外**です。
> - 必ず利益が出るような「必勝予想」の提供
> - 競馬サイトからのオッズ等の自動スクレイピング
> - 実際の馬券の自動購入

## 技術スタック
- **Backend**: Python 3.11+, FastAPI, SQLite, SQLAlchemy, Alembic
- **Frontend**: React, TypeScript, Vite

## 主な機能 (Frontend MVPでできること)
MVPフェーズの現在、ブラウザ画面上から以下の一連の操作フローが利用可能です。
- 収支ダッシュボード表示（全体の投資額、払戻総額、Profit、ROI）
- レース一覧表示および新規登録
- レース選択（選択したレースの情報を下部にドリルダウン表示）
- 出走馬一覧表示および新規登録
- 予想一覧表示および新規登録（順位から予想印への自動変換）
- 馬券購入履歴表示および新規登録
- レース結果表示および新規登録
- **連動自動更新**: レース結果等を登録後、ページリロードなしで画面最上部の収支ダッシュボード（Profit・ROI等）が自動で再計算・更新されます。

## セットアップと起動手順

バックエンドとフロントエンドを別々に立ち上げる必要があります。

### 1. バックエンドの起動
```bash
cd backend
python -m venv .venv
source .venv/Scripts/activate  # Windows の場合 (Mac/Linux の場合は source .venv/bin/activate)
pip install -r requirements.txt

# 初回のみ：マイグレーションと初期データの投入
alembic upgrade head
python seed.py

# 開発用サーバーの起動
uvicorn app.main:app --reload
```
起動後、バックエンドは `http://localhost:8000` で動作します。
Swagger UI は `http://localhost:8000/docs` で確認できます。

### 2. フロントエンドの起動
```bash
cd frontend
npm install
npm run dev
```
起動後、フロントエンドは `http://localhost:5173` で動作します。
ブラウザで上記URLにアクセスするとアプリを利用できます。

> [!NOTE]
> フロントエンドからのAPIリクエスト（`/api/*`）は、Viteのプロキシ設定により自動的にバックエンド（`http://localhost:8000`）へ転送されます。

## `requests.http` の使い方
VS Code拡張機能「REST Client」用のテストファイル `backend/requests.http` を同梱しています。APIの動作確認を手軽に行うことができます。

## MVPの制限事項と今後の拡張予定

### 現在の制限事項
- **UIの実装状況**: レース登録時の会場、馬券購入時の券種、予想登録時の出走馬は、マスタ/選択中レースのデータに基づくselect入力に対応済みです。一部、レース結果の着順入力などはMVPとしての簡易入力仕様です。
- **券種別回収率の正確な算出**: 券種別の正確な払戻・回収率は未対応です。現在、払戻金はレース単位で管理しており、現状はレース単位の `total_return` で全体収支を算出している暫定仕様です（`/api/v1/stats/by-bet-type` の払戻金は一律 `0` となっています）。
- **認証なし**: ユーザー登録やログイン機能はありません。

### 今後の拡張予定
- 登録データの編集・削除（Edit / Delete）UIの追加
- `result_returns` テーブルの実装と、それに基づく券種別払戻の管理
- アプリケーションの Docker 化
- AWS等クラウド環境へのデプロイ
